Welcome!

Accessible University (AU) is a fictional university, and this is its fictional home page. This page is designed to demonstrate a variety of common web design problems that result in visitors with disabilities being unable to access the content or features of a web page. Although the heading at the top of this section of the page suggests that AU may be a welcoming institution, the inaccessible design of this page sends the opposite message.

Bienvenido!

Accesible Universidad (UA) es una universidad ficticia, y esta es su página de ficción. Esta página está diseñada para demostrar una variedad de común de diseño web que se traducen en problemas de los visitantes con discapacidad no poder acceder al contenido o las características de una página web. Aunque el título en la parte superior de esta sección de la página sugiere que AU puede ser una institución de bienvenida, el diseño inaccesible de esta página envía el mensaje opuesto.

    List of Accessibility Issues
    • Missing lang attribute on <html>
    • No ARIA or HTML5 landmarks like <main>, <nav>, <header>
    • No <h1>; headings are visually styled but semantically incorrect
    • Event dates incorrectly marked as headings
    • Low contrast text in footer, nav, buttons, and forms
    • Viewport prevents zooming or scaling
    • No "Skip to main content" link
    • Logo and links lack accessible names
    • Navigation menus not keyboard operable, open only on hover
    • Search input uses placeholder as label, and button is unlabeled
    • Images missing or using bad alternative text
    • Carousel lacks roles, keyboard focus, or visual state indicators
    • Carousel controls have poor contrast, no accessible names
    • Text embedded in images, unreadable at zoom
    • Focus indicators removed using outline: none
    • Links and buttons rely on color alone
    • Ambiguous "click here" links and buttons
    • Interactive elements have no visible focus
    • Form fields not associated with labels
    • Required fields not marked visually or programmatically
    • No autocomplete attributes for personal fields
    • No feedback when submitting invalid forms
    • CAPTCHA is visual-only
    • Checkboxes not grouped or labeled
    • Error messages not announced or associated with fields
    • Empty <div> with invalid ARIA used
    • Footer links like “FB” and “TW” are not descriptive
    • Empty Twitter links in the footer
    • Decorative images have alt text like "horizontal line"
    • <hr> elements are not hidden from screen readers
    • Footer list structure is broken (nested links not inside <li>)
    • Page layout breaks at 200% zoom
    • Mobile users must scroll both horizontally and vertically
    • Mobile menu toggle not keyboard operable or labeled
    • Autocomplete field has no ARIA roles or feedback
    • Newsletter form has disappearing labels, poor contrast, and no error handling
    • Autoplaying hero video cannot be paused or described
    • Dialog itself has no role, heading, label, or focus trap
    • Block links wrap full cards and use redundant labels
    • Use of positive tabindex values causes illogical keyboard focus order
    • DOM order does not match visual layout, confusing screen reader users
    • aria-hidden is misused and not removed when modals close
    • Tabs, filters, and disclosures lack ARIA state indicators like aria-expanded, aria-current
    • Disclosure toggles use non-semantic elements like <div> instead of buttons
